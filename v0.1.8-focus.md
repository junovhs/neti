# v0.1.8 Focus

## North Star

Neti is not just a scanner. It is a professional confidence gate.

When `neti scan` or `neti check` shows all green, that must mean:

- the code compiles
- the formatter ran and is clean
- the linter passed
- the tests ran and passed
- Neti's own governance layer sees no meaningful architecture or quality concerns
- the result is credible enough to show to a client, reviewer, or senior developer without embarrassment
- the code has cleared a serious quality gate and is worth professional review

This is the product contract.

## Meaning Of Green

Green must mean professional confidence, not merely absence of parser errors.

That implies:

- false positives must be very low
- green must be rare enough to mean something
- anything shown as red must be genuinely defensible
- preference-level guidance must not be mixed with hard blockers as though they are the same kind of truth
- the scanner must optimize for trust, not output volume

Operationally:

- `green` = professionally presentable
- `yellow` = advisory improvement opportunities
- `red` = real issues that undermine confidence

Green is not a proof of correctness or security.
Green means the code cleared the quality gate Neti is responsible for.

## Product Identity

Neti is an architecture-governance scanner focused on:

- modularity
- maintainability
- locality
- changeability
- resistance to AI spaghetti code

It is not primarily a generic bug finder.

Security, safety, correctness, and performance checks matter, but they support the core mission rather than define it.

## Security Posture

Neti must never claim to guarantee that code is secure.

No serious static tool can honestly promise absence of vulnerabilities across real-world software, especially across multiple languages and AI-generated code.

The defensible claim is narrower and stronger:

- Neti reduces obvious and structural security risk
- Neti detects known high-signal insecure patterns
- Neti enforces code shape that makes security review easier
- Neti makes AI-generated code more reviewable and less likely to hide obvious danger

Therefore:

- `green` does not mean "provably secure"
- `green` does mean "no known high-confidence security findings detected by Neti, along with no major build, test, lint, or architecture-governance failures"

Neti is a confidence amplifier, not a formal security proof system.

Its role is to turn code from "arbitrary AI sludge" into code that has passed a serious quality gate and is worth professional review.

## Current Reality

Neti is strongest on Rust today.

Current state:

- Rust support is substantial and credible
- non-Rust support exists, but is still narrower and less uniform
- the clearest shared cross-language detections today are the shared-semantic paths such as `P06` and `L02`
- structural metrics, semantic checks, concurrency checks, and many risk checks are still primarily Rust-centric

This means the honest framing is:

Neti is currently a strong Rust-first architectural scanner with an early cross-language semantic engine.

## Severity Discipline

Neti must distinguish between different kinds of signal.

- objective defect or risk
- maintainability signal
- architectural policy
- house-style preference

If a finding is not broadly defensible, it must not be presented as a hard failure by default.

This matters especially for architectural guidance such as file size, cohesion, coupling, and related governance signals. These may still be important, but they must be labeled honestly and calibrated carefully.

## Law Of Atomicity Reframing

`LAW OF ATOMICITY` is currently acting as a maintainability and architecture signal:

- files should stay small because smaller units tend to improve modularity
- smaller files are easier to reason about
- smaller files are easier to refactor safely
- smaller files are easier for AI-assisted editing to rewrite cleanly

The core claim is valid.

The mistake is overstating it as though large files are universally "wrong."

The long-term direction is to keep the signal while improving the framing, severity, and naming so it reflects what it actually means: architectural drag, maintainability budget pressure, and AI-edit friction.

## Proprietary Direction

Neti should not compete by merely accumulating more borrowed lint rules.

The strongest proprietary direction is:

1. use `omni-ast` to normalize semantic primitives across languages
2. derive higher-order architecture metrics from those primitives
3. validate those metrics empirically against real repositories and maintenance outcomes
4. keep the metrics interpretable enough that users can trust and act on them

`Locality` is the model for this approach:

- it is not just a raw lint
- it is not just a direct AST match
- it is a higher-order architectural KPI derived from underlying facts
- it expresses something Neti can own

This should become a broader pattern.

## Proprietary Metric Thesis

Neti-native value will come from composite signals that answer questions like:

- how hard is this code to safely change?
- how much hidden coupling does this module create?
- how blurry are the boundaries between responsibilities?
- how much AI-rewrite friction exists in this file or subsystem?
- how much semantic drift exists between names and behavior?
- how much architecture rot is accumulating even if the code still "works"?

Novelty alone is not enough.

Any proprietary metric must be:

- precisely defined
- reproducible
- interpretable
- empirically validated

## Research Direction

The key research question is not:

"What else can we scan?"

It is:

"What cross-language architectural signals most reliably predict modular, maintainable, AI-resistant code shape?"

That should guide future work.

## v0.1.8 Focus

For this phase, Neti should focus on:

- preserving trust in green status
- reducing false positives
- clarifying severity semantics
- honestly describing Rust-first versus partial cross-language coverage
- strengthening `omni-ast` as the basis for cross-language semantics
- inventing and validating new Neti-native architecture metrics

If Neti says all green, it must mean something real.
